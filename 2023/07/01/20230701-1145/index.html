宣告 let、const、var
Nora
  2023-07-01 11:45:00   2024-08-31 18:37:01  
  • JavaScript
  • let
  • const
  • var
  

未宣告的變數會變成全域 (window) 屬性，這樣別的函式也能呼叫它

會造成全域環境的作用域汙染，之後除錯會變得很困難


宣告的變數會被限制它的作用域

宣告的方法有 let 、 const 和 var


作用域因宣告的語法而有不同

var 為函式作用域(function scope)

let 、 const 的作用域為區塊作用域(block scope)

這三個有點像又有點不太一樣，差異點分別來說明


var

用 var 宣告過的變數可以修改

不過現在已經不太建議使用了

因為用 var 宣告的變數會變成全域變數，會影響其他的程式碼


用 var 宣告的變數

如果在 function 內，則只能在那個 function 裡面使用

稱為區域變數

如果是在 function 外，則在任何位置都可以被呼叫

稱為全域變數


但是如果是在 for 迴圈內用 var 宣告的變數

卻可以被 for 迴圈以外的位置呼叫

必須要改成 let 才會是區域變數的效果


let

用 let 宣告過的變數可以修改

像是 let a = 1; 之後要改成 a = 2; 也是可以的

let 可以修改既有的 Value 的值

用 let 宣告變數會採用區塊作用域來避免產生全域變數


const

用法和 let 差不多

只是 const 在單純的基本型別比較難被修改

像是 const a = 1; 之後要改成 a = 2; 就沒辦法了

所以一些比較不會更動到的東西，就可以用 const


宣告變數的注意事項

  • 開頭不能是數字

  • 大小寫是不一樣的

像是 appleNum 和 applenum 是不一樣的兩個變數

  • 關鍵字(保留字)不能當變數名稱 => 詳細介紹

像是 if for true 之類的
