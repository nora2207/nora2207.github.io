To Do List 待辦事項
Nora
  2024-04-17 23:06:00   2024-09-02 22:05:22  
  • JavaScript
  

版面介紹

這個 TODO LIST 上方可以輸入代辦事項，按右邊的+可以新增

下面分成三個區塊

全部：所有(待完成和已完成)的代辦事項都會在這裡

還沒完成的會是框框，點選之後會變成打勾的圖案，而且文字也會劃掉

待完成：還沒完成的代辦事項會在這裡(框框)

已完成：已經完成的代辦事項會在這裡(打勾)

最下面會顯示現在還有幾個待完成項目

右下的”清除已完成項目”點下去後，已完成的項目都會清除掉，只會剩下待完成的項目




接下來要開始來製作了！

Html和CSS部分使用六角學院的版型

所以大概介紹一下，比較著重於JS的部分


Html

這個是上方待辦事項輸入區

右邊的新增按鈕是直接用 + 的 a 連結來設計



然後是下方清單區


CSS

CSS 部分是用SCSS來寫

定義各個顏色


背景斜斜的顏色區塊


JS

接著要介紹JS的部分

首先先定義上方輸入區為 inputText

右邊的 + 新增按鈕為 addBtn

然後再定義一個 todoData 空陣列來放代辦事項


1.註冊監聽點擊按鈕事件

監聽 + 按鈕，在點擊時

如果輸入欄有輸入內容，就會把這個內容放到清單的最前面

並把剛剛輸入的文字清除

接著再執行 updateList 函式(5.)


2.渲染資料到下方清單

render 函式用來渲染資料

把組好的資料拿來跑 forEach

將輸入的內容組合到 li 中，並放到下方清單內


3.切換分頁 (全部、待完成、已完成)

點擊全部、待完成、已完成會自動套用樣式(黑底線)

一開始是在全部，執行 changeTab 函式來切換樣式

接著再執行 updateList 函式(5.)


4.刪除、切換 checked 狀態

執行 checkDelete 函式來核取/取消代辦事項前方框框及執行右方刪除鍵

每筆代辦事項都有自己的 ID，利用 ID 來判斷是哪一筆資料

接著再執行 updateList 函式(5.)


5.更新代辦清單

用 filter 來篩選是否完成，並計算待完成的項目數量


加上 updateList()

1.新增、3.切換分頁、4.checked 狀態的函式內加上 updateList() 來更新

最外層也加上一個初始的 updateList()


6.清除已完成項目

用 filter 來過濾已完成的項目

也要記得加上 updateList 函式來更新


7.優化(enter新增代辦事項)

鍵盤按 enter 時也可以新增代辦事項


8.增加顏色選擇器

因為在選背景顏色的時候有好幾個顏色都很喜歡

所以就乾脆做一個選擇器，這樣就可以換好幾個顏色了


左上角新增一個按鈕，點下去可以選擇顏色

總共選了五個顏色，每個按鈕都是圓形的

點什麼顏色就會換成那個顏色的背景


切換顯示或是隱藏


設定選到什麼顏色時，就把背景切換成那個顏色
